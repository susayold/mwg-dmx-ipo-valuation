# Methodology documentation

Read these files before changing assumptions or publishing outputs:

1. [`data_scope.md`](data_scope.md) — corporate perimeter, source hierarchy, licensing boundary and publication limitations.
2. [`financial_methodology.md`](financial_methodology.md) — transaction bridge, FY2023–FY2025 three-statement analysis, working-capital/earnings-quality review, DCF/SOTP methodology and validation policy.
3. [`model_spec.md`](model_spec.md) — workbook/data contract, historical bridges, normalization rules, formulas, scenarios, sensitivities and publication gates.

Together they define what counts as an official actual, issuer update, management projection or analyst assumption. The current public module is a **three-statement historical analysis plus Q1 2026 trend update**; it is not presented as a fully integrated three-statement forecast.
