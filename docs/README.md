# Methodology documentation

Read these files before changing assumptions or publishing outputs:

1. [`data_scope.md`](data_scope.md) — corporate perimeter, source hierarchy, licensing boundary and publication limitations.
2. [`financial_methodology.md`](financial_methodology.md) — transaction bridge, financial analysis, DCF/SOTP methodology and validation policy.
3. [`model_spec.md`](model_spec.md) — workbook/data contract, formulas, scenarios, sensitivities and publication gates.

Together they define what counts as an official actual, issuer update, management projection or analyst assumption.
